# Ejercicio Nro: 5

## Enunciado
Proponga cinco elementos que ayuden en el desarrollo de software referido a la propiedad de flexibilidad que se requiere. 

## Resolución
1) Diseño para el cambio: Consiste en estructurar el software pensando desde el inicio que los requisitos van a evolucionar, permitiendo que las modificaciones se realicen de forma controlada y sin efectos laterales impredecibles.

2) Ocultamiento de la Información: Es una técnica para ocultar los detalles de implementación de un módulo detrás de una interfaz clara, de modo que si la lógica interna cambia, el resto del sistema no se ve afectado.

3) Herramientas de especificación: El uso de diagramas (como los de ICONIX o UML) permite que el equipo y el cliente visualicen y refinen lo que el sistema debe hacer de forma iterativa, facilitando ajustes antes de que el código sea demasiado complejo.

4) Desarrollo Iterativo e Incremental: Al producir versiones funcionales frecuentemente, el sistema puede reajustarse continuamente según el feedback del usuario, permitiendo cambios de dirección en el desarrollo de manera flexible.

5) Refactorización constante: Es la práctica de mejorar la estructura interna del código sin cambiar su comportamiento externo, eliminando la deuda técnica y asegurando que el software siga siendo entendible y fácil de modificar a medida que crece.
