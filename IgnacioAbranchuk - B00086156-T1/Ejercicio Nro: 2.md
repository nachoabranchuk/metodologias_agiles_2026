# Ejercicio Nro: 2

## Enunciado
¿Qué es para vosotros el desarrollo de software?
* ¿Ejemplos de desarrollo de software en los que hayas participado? ¿Ejemplos que conozcas?
* ¿Cuál es el papel del desarrollador en el desarrollo de un proyecto software?
* ¿Hay otros papeles importantes en el desarrollo, aparte del desarrollador?
* ¿Cuándo se ha desarrollado un buen software? 

## Resolución
* Participé en un proyecto/trabajo de campo en las materias Ingeniería de Software y Trabajo de Diploma. Fuera de eso, conozco ejemplos cotidianos como las aplicaciones de gestión de billeteras virtuales, que sirven para entender cómo se aplican las reglas de negocio al código.

* El desarrollador es el encargado de traducir los requisitos y modelos conceptuales en código ejecutable y probado. Su rol implica tomar decisiones sobre la implementación de la lógica necesaria para que el sistema funcione.

* Sí, existen muchos roles fundamentales como el Analista de Sistemas (que identifica qué debe hacer el software), el Arquitecto (que define la estructura base), los Testers y el Cliente, que tiene un papel activo validando el valor de lo que se entrega.

* Se considera un buen software cuando cumple con dos cosas: cuando resuelve el problema correcto y cuando está construido correctamente.
